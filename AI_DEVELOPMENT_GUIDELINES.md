# US Macro Tracker: AI Development Guidelines

今後の機能拡張やメンテナンスにおいて、AIアシスタント（エージェント）が同様のバグを引き起こさないための重要方針です。新しいAIセッションを開始する際はこのファイルを必ず参照してください。

## 1. UI・レスポンシブデザインの遵守事項
*   **Tailwindのモバイルファースト原則を厳守すること**: 
    - `text-4xl sm:text-2xl` のような「スマホで巨大化し、PCで縮む」逆転現象の設定は絶対に行わない。
    - スマホでの文字切れを防ぐため、ヘッダー等の折り返せない文字には必ず `truncate` を使用し、文字サイズは `text-xl sm:text-2xl md:text-3xl` のように滑らかに拡大させること。
*   **既存の安定したUIの破壊禁止**: 新機能（ダークモード切り替えボタン等）を追加する際、既存の文字レイアウトやPaddingを無理に崩さないこと。

## 2. FRED API データ取得の遵守事項
*   **正しい指標IDの選択**: 個人消費支出のデフレーターは `PCE`（総額）ではなく、必ず `PCEPI`（物価指数）を使用すること。
*   **パーセント変化率の必須化（unitsパラメータ）**: 
    - 消費者物価指数（CPI）、生産者物価指数（PPI）などのインフレ率は、そのまま取得すると「単なる指数（300等）」が返る。そのため必ずAPIパラメータに `&units=pc1`（前年同月比）を含めること。
    - 実質GDPは `pca`（前期比年率）を使用すること。
    - 各指標のフロントエンドでの表示名には `[前年同月比]` や `[前期比年率]` など、単位を明示すること。

## 3. アプリ化（Capacitor）の更新フロー
モバイルアプリ版（iOS/Android）の土台はすでに `ios/` および `android/` フォルダに構築済みです。
機能追加後は、必ず以下のコマンドでWeb版のビルドとアプリ側への同期を行うこと。
```bash
npm run build
npx cap sync
```

## 4. 環境移行・バックアップに関する共通認識
*   **コードの完全保全**: 本プロジェクトの全ソースコードと履歴は常にGitHub上にリモート保存（Push）されており、作業環境（PC）の破損や買い替えが発生しても、リポジトリを `git clone` するだけで完全に復旧可能である。
*   **.env.local の扱い**: セキュリティの観点から `.env.local` （FRED_API_KEY等の秘密情報）はGitHubのアップロード対象外（`.gitignore`）となっている。そのため、新しいPCや別環境で開発を再開する際は、必ずルートディレクトリに `.env.local` を新規作成し、正しいAPIキーを手動で設定してから開発・ビルドを行うこと。
